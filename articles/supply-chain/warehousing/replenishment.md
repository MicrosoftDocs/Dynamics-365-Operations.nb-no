# <a name="replenishment-overview"></a><span data-ttu-id="65eca-103">Oversikt over etterfylling</span><span class="sxs-lookup"><span data-stu-id="65eca-103">Replenishment overview</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="65eca-104">Dette emnet beskriver etterfyllingsstrategiene som er tilgjengelige for lagre som bruker funksjonaliteten som er tilgjengelig i Lagerstyring.</span><span class="sxs-lookup"><span data-stu-id="65eca-104">This topic describes the replenishment strategies that are available for warehouses that use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="65eca-105">Informasjonen i dette emnet gjelder ikke for lagrerløsningen som er tilgjengelig i Lagerstyring.</span><span class="sxs-lookup"><span data-stu-id="65eca-105">The information in this topic doesn't apply to the warehousing solution that is available in Inventory management.</span></span>

<span data-ttu-id="65eca-106">Følgende etterfyllingsstrategier er tilgjengelige:</span><span class="sxs-lookup"><span data-stu-id="65eca-106">The following replenishment strategies are available:</span></span>

- <span data-ttu-id="65eca-107">**Etterfylling basert på bølgebehov**– Denne strategien oppretter etterfyllingsarbeid for utgående ordrer eller laster, hvis beholdning ikke er tilgjengelig når bølgen oppretter arbeidet.</span><span class="sxs-lookup"><span data-stu-id="65eca-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="65eca-108">Etterfyllingsarbeid kan for eksempel opprettes hvis antallet som kreves for en salgsordre ikke er tilgjengelig når en bølge behandles.</span><span class="sxs-lookup"><span data-stu-id="65eca-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
- <span data-ttu-id="65eca-109">**Etterfylling basert på minimums- og maksimumsantall** – Denne strategien bruker minimums og maksimumsgrenser for lagring til å bestemme når lokasjoner skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="65eca-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="65eca-110">Vare- og lokasjonskriteriene definere beholdningen som evalueres for etterfylling.</span><span class="sxs-lookup"><span data-stu-id="65eca-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="65eca-111">Maler for etterfylling basert på minimums- og maksimumsantall er den primære mekanismen for å opprettholde optimal nivåer på plukklokasjoner.</span><span class="sxs-lookup"><span data-stu-id="65eca-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="65eca-112">Du kan bruke behovsetterfylling for å garantere at nok pålydende plukking av lager er tilgjengelig for å dekke bølgeetterspørselen, som et supplement mellom sykluser for etterfylling basert på minimums- og maksimumsantall.</span><span class="sxs-lookup"><span data-stu-id="65eca-112">To help guarantee that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
- <span data-ttu-id="65eca-113">**Etterfylling basert på lastbehov** – Denne strategien summerer behovet for flere laster og oppretter etterfyllingsarbeidet som kreves for å fylle de relevante plukklokasjonene.</span><span class="sxs-lookup"><span data-stu-id="65eca-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="65eca-114">Denne strategien bidrar til å garantere at lastene som opprettes, kan plukkes på lageret når de er frigitt.</span><span class="sxs-lookup"><span data-stu-id="65eca-114">This strategy helps guarantee that the loads that are created can be picked in the warehouse after they are released.</span></span>
- <span data-ttu-id="65eca-115">**Umiddelbar etterfylling** – Denne strategien etterfyller beholdningen før en bølge kjøres hvis allokering mislykkes for en stedsdirektivlinje som har en etterfyllingsmal.</span><span class="sxs-lookup"><span data-stu-id="65eca-115">**Immediate replenishment** – This strategy replenishes inventory before a wave is run if allocation fails for a location directive line that has a replenishment template.</span></span> 

<span data-ttu-id="65eca-116">Alle fire strategiene oppretter etterfyllingsarbeid, basert på en etterfyllingsmal.</span><span class="sxs-lookup"><span data-stu-id="65eca-116">All four strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="65eca-117">Etterfylling basert på bølgebehov</span><span class="sxs-lookup"><span data-stu-id="65eca-117">Wave demand replenishment</span></span>
<span data-ttu-id="65eca-118">Etterfylling basert på bølgebehov oppretter etterfyllingsarbeid basert på behov, hvis mengden som kreves for produksjonsordrer, kanbans, utgående ordrer eller laster ikke er tilgjengelige når en bølge oppretter arbeid.</span><span class="sxs-lookup"><span data-stu-id="65eca-118">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders, or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="65eca-119">Etterfyllingsmalen inneholder informasjon om varekriteriene, måleenhet, behovsintervall og lokasjon.</span><span class="sxs-lookup"><span data-stu-id="65eca-119">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="65eca-120">Lokasjonsdirektiver brukes til å fastsette hvilken lokasjon som skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="65eca-120">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="65eca-121">Du koble disse lokasjonsdirektivene til etterfyllingsmalen ved hjelp av feltet **Direktivkode**.</span><span class="sxs-lookup"><span data-stu-id="65eca-121">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="65eca-122">Hvis feltet **Direktivkode** er ikke satt, brukes spørringer til å bestemme hvilket lokasjonsdirektiv som skal brukes.</span><span class="sxs-lookup"><span data-stu-id="65eca-122">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="65eca-123">Vær oppmerksom på at hvis en direktivkode ikke er angitt i etterfyllingsmalen, og lokasjonsdirektivet har en direktivkode, ignoreres direktivlokasjonen selv om spørringen i lokasjonsdirektivet er riktig.</span><span class="sxs-lookup"><span data-stu-id="65eca-123">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="65eca-124">Lokasjonsdirektiver for plukking brukes til å avgjøre hvor beholdning hentes fra for etterfyllingen.</span><span class="sxs-lookup"><span data-stu-id="65eca-124">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="65eca-125">I tillegg til å opprette en mal, må du angi noen innstillinger for etterfylling i bølgemalen.</span><span class="sxs-lookup"><span data-stu-id="65eca-125">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="65eca-126">Denne bølgemalen bør inneholde et bølgetrinn for etterfylling, som kjøres kun hvis et element ikke er vellykket allokert.</span><span class="sxs-lookup"><span data-stu-id="65eca-126">The wave template should contain a wave step for replenishment that is run only if an item isn't successfully allocated.</span></span> <span data-ttu-id="65eca-127">Dette bølgetrinnet for etterfylling bruker en bølgetrinnkode for å avgjøre hvilken etterfyllingsmal som skal brukes.</span><span class="sxs-lookup"><span data-stu-id="65eca-127">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="65eca-128">I tillegg til å ha bølgetrinn for etterfylling, må du forsikre deg om at **Etterfylling** er valgt i delen **Metoder** for bølgemalen.</span><span class="sxs-lookup"><span data-stu-id="65eca-128">In addition to having a wave step for replenishment, you must make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span>

<span data-ttu-id="65eca-133">**Etterfyllingsenhet** er minimumsenheten som skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="65eca-133">**Replenishment unit** is the minimum unit to replenish.</span></span> <span data-ttu-id="65eca-134">Dette må være et heltall som er et multiplum av enheten.</span><span class="sxs-lookup"><span data-stu-id="65eca-134">This must be a whole number that is a multiple of the unit.</span></span> <span data-ttu-id="65eca-135">Systemet runder opp til den høyeste enheten som er mulig når du oppretter arbeid.</span><span class="sxs-lookup"><span data-stu-id="65eca-135">The system will round up to the highest unit possible when creating work.</span></span>

<span data-ttu-id="65eca-136">Etterfylling av behov støttes for salgsordrer, overføringsordrer, produksjonsordrer og Kanbaner.</span><span class="sxs-lookup"><span data-stu-id="65eca-136">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="65eca-137">Etterfylling basert på minimums- og maksimumsantall</span><span class="sxs-lookup"><span data-stu-id="65eca-137">Min/Max replenishment</span></span>
<span data-ttu-id="65eca-138">I etterfylling basert på minimums- og maksimumsantall etterfylles lageret slik at det er mellom minimums- og maksimumsgrensene som er angitt.</span><span class="sxs-lookup"><span data-stu-id="65eca-138">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="65eca-139">Vanligvis skjer denne prosessen en gang hver dag for å sikre at alle plukklokasjoner fylles til maksimumsnivået før plukkingen starter.</span><span class="sxs-lookup"><span data-stu-id="65eca-139">Typically, this process occurs one time every day, to help guarantee that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="65eca-140">Minimums- og maksimumsantallet angis i en etterfyllingsmal.</span><span class="sxs-lookup"><span data-stu-id="65eca-140">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="65eca-141">Mange av de andre innstillingene i malen ligner på innstillinger i malene som brukes etterfylling basert på bølgebehov.</span><span class="sxs-lookup"><span data-stu-id="65eca-141">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="65eca-142">Malen må inneholde én linje for hver vare og lokasjon.</span><span class="sxs-lookup"><span data-stu-id="65eca-142">The template should contain one line for each item and location.</span></span> <span data-ttu-id="65eca-143">Når du kjører etterfylling ved hjelp av den satsvise jobben, evaluerer systemet om etterfylling kreves, ved å bruke rekkefølgen linjene er organisert i.</span><span class="sxs-lookup"><span data-stu-id="65eca-143">When you run replenishment by using the batch job, the system evaluates whether replenishment is required by using the sequence that the lines are organized in.</span></span> 

<span data-ttu-id="65eca-144">Vær oppmerksom på at strategien for etterfylling basert på minimums- og maksimumsantall ikke kan etterfylle en tom lokasjon med mindre lokasjonen er definert som fast lokasjon for varen.</span><span class="sxs-lookup"><span data-stu-id="65eca-144">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="65eca-145">Hvis lokasjonen som skal etterfylles ikke er en fast lokasjon, kan ikke systemet fastsette hvilket element som skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="65eca-145">If the location that must be replenished isn't a fixed location, the system can't determine which item should be replenished.</span></span> <span data-ttu-id="65eca-146">Derfor kreves det minst noe beholdning før etterfylling oppstår.</span><span class="sxs-lookup"><span data-stu-id="65eca-146">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
