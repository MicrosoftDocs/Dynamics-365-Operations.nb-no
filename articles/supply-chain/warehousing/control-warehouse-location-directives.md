## <a name="work-templates"></a><span data-ttu-id="50193-108">Arbeidsmaler</span><span class="sxs-lookup"><span data-stu-id="50193-108">Work templates</span></span>

<span data-ttu-id="50193-109">Siden **Arbeidsmaler** lar deg definere arbeidsoperasjoner som må utføres på lageret.</span><span class="sxs-lookup"><span data-stu-id="50193-109">The **Work templates** page lets you define the work operations that must be performed in the warehouse.</span></span> <span data-ttu-id="50193-110">Vanligvis består lageroperasjoner av to handlinger: en lagerarbeider plukker opp beholdning på en lokasjon og plasserer den plukkede beholdningen på en annen lokasjon.</span><span class="sxs-lookup"><span data-stu-id="50193-110">Typically, warehouse work operations consist of a pair of actions: a warehouse worker picks up on-hand inventory in one location and then puts the picked inventory down in another location.</span></span> 

<span data-ttu-id="50193-111">Arbeidsmaler består av et hode og tilhørende linjer.</span><span class="sxs-lookup"><span data-stu-id="50193-111">Work templates consists of a header and associated lines.</span></span> <span data-ttu-id="50193-112">Hver arbeidsmal gjelder en bestemt *arbeidsordretype*.</span><span class="sxs-lookup"><span data-stu-id="50193-112">Each work template is for a specific *work order type*.</span></span> <span data-ttu-id="50193-113">Mange arbeidsordretyper assosieres med kildedokumenter, for eksempel bestillinger eller salgsordrer.</span><span class="sxs-lookup"><span data-stu-id="50193-113">Many work order types are associated with source documents, such as purchase or sales orders.</span></span> <span data-ttu-id="50193-114">Imidlertid representerer andre arbeidsordretyper egne lagerprosesser, for eksempel syklustelling.</span><span class="sxs-lookup"><span data-stu-id="50193-114">However, other work order types represent separate warehouse processes, such as cycle counting.</span></span> <span data-ttu-id="50193-115">*Arbeidspulje-ID* gir deg muligheten til å organisere arbeidet i grupper.</span><span class="sxs-lookup"><span data-stu-id="50193-115">The *work pool ID* lets you organize work into groups.</span></span>

<span data-ttu-id="50193-124">Du kan definere en spørring til å kontrollere når en bestemt arbeidsmal brukes.</span><span class="sxs-lookup"><span data-stu-id="50193-124">You can set up a query to control when a particular work template is used.</span></span> <span data-ttu-id="50193-125">Du kan for eksempel angi en begrensning slik at en bestemt mal kan brukes for arbeid bare i et bestemt lager.</span><span class="sxs-lookup"><span data-stu-id="50193-125">For example, you can set a limitation so that a particular template can be used for work only in a specific warehouse.</span></span> <span data-ttu-id="50193-126">Du kan også ha flere maler som brukes til å opprette arbeid for utgående ordrebehandling, avhengig av salgsopprinnelsen.</span><span class="sxs-lookup"><span data-stu-id="50193-126">Alternatively, you might have several templates that are used to create work for outbound sales order processing, depending on the sales origin.</span></span> <span data-ttu-id="50193-127">Systemet bruker **Serienummer**-feltet for å bestemme rekkefølgen som de tilgjengelig arbeidsmalene vurderes i.</span><span class="sxs-lookup"><span data-stu-id="50193-127">The system uses the **Sequence number** field to determine the order that the available work templates are assessed in.</span></span> <span data-ttu-id="50193-128">Hvis du har en bestemt spørring etter en bestemt arbeidsmal, bør du derfor gi den et lavt sekvensnummer.</span><span class="sxs-lookup"><span data-stu-id="50193-128">Therefore, if you have a very specific query for a particular work template, you should give it a low sequence number.</span></span> <span data-ttu-id="50193-129">Denne spørringen vil deretter bli evaluert før andre, mer generelle spørringer.</span><span class="sxs-lookup"><span data-stu-id="50193-129">That query will then be evaluated before other, more general queries.</span></span> 

<span data-ttu-id="50193-130">Hvis du vil stoppe eller pause en arbeidsprosess, kan du bruke innstillingen **Stopp arbeid** for arbeidslinjen.</span><span class="sxs-lookup"><span data-stu-id="50193-130">To stop or pause a work process, you can use the **Stop work** setting on the work line.</span></span> <span data-ttu-id="50193-131">I så fall vil arbeideren som utfører arbeidet, ikke bli bedt om å utføre neste arbeidslinjetrinn.</span><span class="sxs-lookup"><span data-stu-id="50193-131">In that case, the worker who is performing the work won't be asked to perform the next work line step.</span></span> <span data-ttu-id="50193-132">For å flytte til neste trinn må denne arbeideren eller en annen arbeider velge arbeidet på nytt.</span><span class="sxs-lookup"><span data-stu-id="50193-132">To move on to the next step, that worker or another worker must select the work again.</span></span> <span data-ttu-id="50193-133">Du kan også skille oppgavene i et stykke arbeid ved å bruke en annen *arbeidsklasse-ID* på arbeidsmallinjene.</span><span class="sxs-lookup"><span data-stu-id="50193-133">You can also separate the tasks within a piece of work by using a different *work class ID* on the work template lines.</span></span>

## <a name="location-directives"></a><span data-ttu-id="50193-134">Lokasjonsdirektiver</span><span class="sxs-lookup"><span data-stu-id="50193-134">Location directives</span></span>

<span data-ttu-id="50193-138">I hodet må hvert lokasjonsdirektiv må være knyttet til en *arbeidsordretype* som angir typen lagertransaksjon som direktivet som skal brukes for, for eksempel salgsordrer, etterfylling eller råvareplukking.</span><span class="sxs-lookup"><span data-stu-id="50193-138">On the header, each location directive must be associated with a *work order type* that specifies the type of inventory transaction that the directive will be used for, such as sales orders, replenishment, or raw material picking.</span></span> <span data-ttu-id="50193-139">*Arbeidstypen* angir om lokasjonsdirektivet skal brukes for å plukke eller plassere arbeid eller en annen lagerprosess, for eksempel telle eller lagerstatusendringer.</span><span class="sxs-lookup"><span data-stu-id="50193-139">The *work type* specifies whether the location directive will be used for picking or putting work, or for some other warehouse process, such as counting or inventory status changes.</span></span> <span data-ttu-id="50193-140">Du må også angi et *område* og et *lager*.</span><span class="sxs-lookup"><span data-stu-id="50193-140">You must also specify a *site* and a *warehouse*.</span></span> <span data-ttu-id="50193-141">En *direktivkode* som du angir i hodet, kan brukes til å koble lokasjonsdirektivet til én eller flere maler for arbeid.</span><span class="sxs-lookup"><span data-stu-id="50193-141">A *directive code* that you specify on the header can be used to link the location directive to one or more work templates.</span></span> 

<span data-ttu-id="50193-142">Som for arbeidsmaler kan du definere en spørring for å bestemme når et bestemt lokasjonsdirektiv skal brukes.</span><span class="sxs-lookup"><span data-stu-id="50193-142">As for work templates, you can set up a query to determine when a particular location directive is used.</span></span> <span data-ttu-id="50193-143">Du kan for eksempel angi at når e-handel er opprinnelsen til en salgsordre, må beholdning plukkes fra et dedikert område i lageret.</span><span class="sxs-lookup"><span data-stu-id="50193-143">For example, you can specify that when e-commerce is the origin of a sales order, the inventory must be picked up from a dedicated area in the warehouse.</span></span> <span data-ttu-id="50193-144">Systemet bruker **Serienummer**-feltet for å bestemme rekkefølgen som de tilgjengelig lokasjonsdirektivene vurderes i.</span><span class="sxs-lookup"><span data-stu-id="50193-144">The system uses the **Sequence number** field to determine the order that the available location directives are assessed in.</span></span>

<span data-ttu-id="50193-145">Lokasjonsdirektivlinjene setter tilleggsbegrensninger på bruken av reglene for lokasjonssøk.</span><span class="sxs-lookup"><span data-stu-id="50193-145">The location directive lines set additional restrictions on the application of the location finding rules.</span></span> <span data-ttu-id="50193-146">Du kan angi et minimums- og maksimumsantall som direktivet skal gjelde for, og du kan angi at direktivet skal være for en bestemt lagerenhet.</span><span class="sxs-lookup"><span data-stu-id="50193-146">You can specify a minimum quantity and a maximum quantity that the directive should apply to, and you can specify that the directive should be for a specific inventory unit.</span></span> <span data-ttu-id="50193-147">Hvis måleenheten for eksempel er paller, kan varer på paller plasseres på en bestemt lokasjon.</span><span class="sxs-lookup"><span data-stu-id="50193-147">For example, if the unit of measure is pallets, the items in pallets can be put in a specific location.</span></span> <span data-ttu-id="50193-148">Du kan også angi om antallet kan deles på tvers av flere lokasjoner.</span><span class="sxs-lookup"><span data-stu-id="50193-148">You can also specify whether the quantity can be split across multiple locations.</span></span> <span data-ttu-id="50193-149">Som lokasjonsdirektivhodet har hver lokasjonsdirektivlinje et sekvensnummer som bestemmer rekkefølgen som linjene vurderes i.</span><span class="sxs-lookup"><span data-stu-id="50193-149">Like the location directive header, each location directive line has a sequence number that determines the order that the lines are assessed in.</span></span>
