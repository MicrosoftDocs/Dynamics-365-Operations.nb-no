[!include [banner](../includes/banner.md)]

<span data-ttu-id="1e8f3-107">For å generere en behovsprognose går du til **Hovedplanlegging &gt; Prognose &gt; Behovsprognose &gt; Generer statistisk basislinjeprognose**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="1e8f3-108">Prognoseperioden kan velges på tidspunktet for generering av prognosen.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="1e8f3-109">Tilgjengelige verdier er: dag, uke og måned.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="1e8f3-110">Antall perioder å generere en prognose for settes i feltet **Prognosehorisont**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="1e8f3-111">Når prognosestrategien er satt til **Kopier over historisk behov**, ignoreres slutten av den historiske horisonten.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="1e8f3-112">Systemet kopierer antall perioder som er angitt i feltet **Prognosehorisont** for å prognostisere behov, fra og med datoen som er angitt **Fra-dato** feltet under **Historisk horisont**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="1e8f3-113">Når du kopierer historisk behov fra en bestemt dato fremover, kan produksjonsplanleggere gjøre planen for neste kvartal på to måter:</span><span class="sxs-lookup"><span data-stu-id="1e8f3-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="1e8f3-114">Ved å kopiere behov fra samme kvartal i fjor.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="1e8f3-115">Ved å kopiere behov fra forrige kvartal.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="1e8f3-116">For å unngå forvirring i produksjonsplanene, kan et bestemt antall prognoseperioder låses.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="1e8f3-117">Dette nummeret angis **Låsningshorisont** feltet.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="1e8f3-118">På siden **Justert behovsprognose** er cellene for låste perioder deaktivert, for å gi en visuell indikasjon på at disse verdiene ikke skal endres.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="1e8f3-119">Startdatoen for behovsprognose for basislinje behøver ikke være gjeldende dato eller en dato i fremtiden.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="1e8f3-120">Hvis du vil angi en annen startdato, kan du bruke feltet **Startdato for basislinjeprognose - Fra-dato**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="1e8f3-121">I juni, kan brukerne for eksempel generere en prognose for neste år.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="1e8f3-122">Fordi prognoseperioder mellom slutten av historiske behov og starten på basislinjen mangler, er det ikke sikkert at forutsigelser blir nøyaktige.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="1e8f3-123">Hvis du bruker behovsprognoseservice, er det fire måter som du kan fylle ut manglene på.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-123">If you are using the Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="1e8f3-124">Du kan velge metoden du vil ha ved å angi parameteren MISSING\_VALUE\_SUBSTITUTION på siden **Parametere for behovsprognose**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

> [!NOTE]
> <span data-ttu-id="1e8f3-125">Erstatning av manglende verdi fungerer bare for hullene i data mellom start- og sluttdatoene for historiske data.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="1e8f3-126">Den fyller ikke inn data før eller etter det siste fysiske datapunktet, og den fungerer bare som ekstrapolering mellom eksisterende datapunkt.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span> 

<span data-ttu-id="1e8f3-127">Feltet **Startdato for basislinjeprognose** - **Fra-dato** må være satt til begynnelsen av en prognoseperioden, for eksempel i USA en søndag, hvis prognoseperioden er uken.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-127">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="1e8f3-128">Systemet justerer automatisk feltet **Startdato for basislinjeprognose** - **Fra-dato** slik at det samsvarer med begynnelsen av en prognoseperiode.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-128">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="1e8f3-129">Feltet **Startdato for basislinjeprognose** - **Fra-dato** kan settes til en dato i fortiden.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-129">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="1e8f3-130">Det er altså mulig å generere en behovsprognose i fortid.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-130">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="1e8f3-131">Dette er nyttig fordi du kan la brukere justere serviceparameterne for prognose slik at den statistiske prognosen som er generert tidligere, samsvarer med det faktiske historiske behovet.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-131">This is useful, because it lets users adjust the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="1e8f3-132">Brukere kan deretter fortsette å bruke disse parameterinnstillingene til å generere en statistisk basislinjeprognose for fremtiden.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-132">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="1e8f3-133">Manuelle justeringer i tidligere behovsprognose gjentakelser kan brukes automatisk på den nye basislinjeprognosen hvis den boksen **Overfør manuelle justeringer til baseline forecast if the **Transfer manual adjustments to the demand forecas**t check box is selected.</span></span> <span data-ttu-id="1e8f3-134">Hvis merket er fjernet, legges ikke de manuelle justeringene til i basislinje prognosen, men de slettes ikke.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="1e8f3-135">Manuelle justeringer i en prognose kan slettes bare ved prognose importen, ved å deaktivere den boksen **Lagre tidspunktet for godkjenning.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-136">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="1e8f3-137">Hvis en bruker gjør manuelle justeringer i prognosen, men ikke godkjenner prognosen tilbake til Supply Chain Management, går derfor endringene tapt.</span><span class="sxs-lookup"><span class="sxs-lookup"><span data-stu-id="1e8f3-138">For more information about manual adjustments and how they work, see [Authorize an adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="1e8f3-139">En behovsprognosegenerering kan ha et navn og merknader som hjelper brukere med å identifisere prognosen som genereres.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="1e8f3-140">Disse verdiene vises i prognosens genereringshistorikk på siden **Genereringshistorikk for statistisk basislinjeprognose**.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span>

> <span data-ttu-id="1e8f3-144">Brukere kan noen ganger motta feilmeldinger mens de genererer en behovsprognose, eller prognosegenerering kan fullføres uten øktlogg.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-144">Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="1e8f3-145">Dette kan skje på grunn av overflødige data i spørringen som tidligere ble brukt for generering av prognose.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-145">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="1e8f3-146">For å løse dette problemet klikker du på **Velg** for å åpne **Spørring**-siden, velger **Tilbakestill** og genererer deretter den basislinjeprognosen på nytt.</span><span class="sxs-lookup"><span data-stu-id="1e8f3-146">To resolve this issue, click **Select** to open the **Query** page, select **Reset**, and then regenerate the baseline forecast.</span></span>
