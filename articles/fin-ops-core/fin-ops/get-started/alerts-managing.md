<span data-ttu-id="eb4e6-120">På den annen side kan en satsvis jobb som oppstår sjeldnere, og som er planlagt på tider når systembelastningen er lav, forbedre systemytelsen.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="eb4e6-121">En lav hyppighet for satsvis behandling kan imidlertid ikke oppfyller brukernes behov for betimelige varsler.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="eb4e6-122">Når du konfigurerer hyppigheten for satsvis behandling for endringsbasert hendelser, må du derfor vurdere balansen mellom betimeligheten for varslene og ytelsen til hele systemet.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-122">Therefore, when you set up the frequency of batch processing for change-based events, consider the balance between the timeliness of alerts and the performance of the whole system.</span></span> <span data-ttu-id="eb4e6-123">Disse hensynene blir mer relevante jo større antall brukere som oppretter varslingsregler.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-123">These considerations become more relevant as the number of users who create alert rules increases.</span></span> <span data-ttu-id="eb4e6-124">Hyppigheten påvirker ikke hvor mange hendelser som må behandles.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-124">The frequency doesn't affect the number of events that must be processed.</span></span> <span data-ttu-id="eb4e6-125">Hvis flere brukere imidlertid oppretter regler, må flere kontroller utføres.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-125">However, if more users create rules, more checks must be done.</span></span> <span data-ttu-id="eb4e6-126">Denne typen datautveksling kan påvirke systemets ytelse.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-126">This type of data exchange might affect system performance.</span></span>

<span data-ttu-id="eb4e6-128">Hvis du konfigurerer lav hyppighet for satsvis behandling for endringsbaserte hendelser, kan det hende data som er relevante for betingelsene i varslingsregler, blir endret før partiet er behandlet.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="eb4e6-129">Derfor kan du miste varsler.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="eb4e6-130">En varslingsregel er for eksempel konfigurert til å utløse et varsel når **kundekontakten endres**, og betingelsen er at **kunden = BB**.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="eb4e6-131">Med andre ord, når kundekontakten for kunde BB endres, logges hendelsen.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="eb4e6-132">Systemet for satsvis behandling er imidlertid konfigurert slik at satsvis behandling forekommer sjeldnere enn dataregistrering.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="eb4e6-133">Hvis kundenavnet endres fra **BB** til **AA** før hendelsen behandles, oppfyller ikke dataene i databasen lenger betingelsen i regelen, **kunde = BB**.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="eb4e6-134">Når hendelsen behandles til slutt, genereres det derfor ikke noe varsel.</span><span class="sxs-lookup"><span data-stu-id="eb4e6-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
