# <a name="replenishment"></a><span data-ttu-id="efc6d-103">Etterfylling</span><span class="sxs-lookup"><span data-stu-id="efc6d-103">Replenishment</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="efc6d-104">Dette emnet beskriver etterfyllingsstrategier som er tilgjengelige for lagre som bruker funksjoner som er tilgjengelige i Lagerstyring.</span><span class="sxs-lookup"><span data-stu-id="efc6d-104">This topic describes the replenishment strategies that are available for warehouses which use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="efc6d-105">Informasjonen gjelder ikke for datalagerstyringsløsningen som er tilgjengelig i Lagerstyring.</span><span class="sxs-lookup"><span data-stu-id="efc6d-105">The information doesn't apply to the warehousing solution that is available in Inventory management.</span></span> <span data-ttu-id="efc6d-106">Følgende etterfyllingsstrategier er tilgjengelige:</span><span class="sxs-lookup"><span data-stu-id="efc6d-106">The following replenishment strategies are available:</span></span>

-   <span data-ttu-id="efc6d-107">**Etterfylling basert på bølgebehov**– Denne strategien oppretter etterfyllingsarbeid for utgående ordrer eller laster, hvis beholdning ikke er tilgjengelig når bølgen oppretter arbeidet.</span><span class="sxs-lookup"><span data-stu-id="efc6d-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="efc6d-108">Etterfyllingsarbeid kan for eksempel opprettes hvis antallet som kreves for en salgsordre ikke er tilgjengelig når en bølge behandles.</span><span class="sxs-lookup"><span data-stu-id="efc6d-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
-   <span data-ttu-id="efc6d-109">**Etterfylling basert på minimums- og maksimumsantall** – Denne strategien bruker minimums og maksimumsgrenser for lagring til å bestemme når lokasjoner skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="efc6d-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="efc6d-110">Vare- og lokasjonskriteriene definere beholdningen som evalueres for etterfylling.</span><span class="sxs-lookup"><span data-stu-id="efc6d-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="efc6d-111">Maler for etterfylling basert på minimums- og maksimumsantall er den primære mekanismen for å opprettholde optimal nivåer på plukklokasjoner.</span><span class="sxs-lookup"><span data-stu-id="efc6d-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="efc6d-112">Du kan bruke behovsetterfylling for å sikre at nok pålydende plukking av lager er tilgjengelig for å dekke bølgeetterspørselen, som et supplement mellom sykluser for etterfylling basert på minimums- og maksimumsantall.</span><span class="sxs-lookup"><span data-stu-id="efc6d-112">To ensure that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
-   <span data-ttu-id="efc6d-113">**Etterfylling basert på lastbehov** – Denne strategien summerer behovet for flere laster og oppretter etterfyllingsarbeidet som kreves for å fylle de relevante plukklokasjonene.</span><span class="sxs-lookup"><span data-stu-id="efc6d-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="efc6d-114">Denne strategien bidrar til å sikre at lastene som opprettes, kan plukkes på lageret når de er frigitt.</span><span class="sxs-lookup"><span data-stu-id="efc6d-114">This strategy helps to ensure that the loads that are created can be picked in the warehouse after they are released.</span></span>

<span data-ttu-id="efc6d-115">Alle tre strategier opprette etterfyllingsarbeid basert på en etterfyllingsmal.</span><span class="sxs-lookup"><span data-stu-id="efc6d-115">All three strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="efc6d-116">Etterfylling basert på bølgebehov</span><span class="sxs-lookup"><span data-stu-id="efc6d-116">Wave demand replenishment</span></span>

<span data-ttu-id="efc6d-117">Etterfylling basert på bølgebehov oppretter etterfyllingsarbeid basert på behov, hvis antallet som kreves for produksjonsordrer, karbaner, utgående ordrer eller laster ikke er tilgjengelig når en bølge oppretter arbeid.</span><span class="sxs-lookup"><span data-stu-id="efc6d-117">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="efc6d-118">Etterfyllingsmalen inneholder informasjon om varekriteriene, måleenhet, behovsintervall og lokasjon.</span><span class="sxs-lookup"><span data-stu-id="efc6d-118">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="efc6d-119">Lokasjonsdirektiver brukes til å fastsette hvilken lokasjon som skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="efc6d-119">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="efc6d-120">Du koble disse lokasjonsdirektivene til etterfyllingsmalen ved hjelp av feltet **Direktivkode**.</span><span class="sxs-lookup"><span data-stu-id="efc6d-120">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="efc6d-121">Hvis feltet **Direktivkode** er ikke satt, brukes spørringer til å bestemme hvilket lokasjonsdirektiv som skal brukes.</span><span class="sxs-lookup"><span data-stu-id="efc6d-121">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="efc6d-122">Vær oppmerksom på at hvis en direktivkode ikke er angitt i etterfyllingsmalen, og lokasjonsdirektivet har en direktivkode, ignoreres direktivlokasjonen selv om spørringen i lokasjonsdirektivet er riktig.</span><span class="sxs-lookup"><span data-stu-id="efc6d-122">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="efc6d-123">Lokasjonsdirektiver for plukking brukes til å avgjøre hvor beholdning hentes fra for etterfyllingen.</span><span class="sxs-lookup"><span data-stu-id="efc6d-123">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="efc6d-124">I tillegg til å opprette en mal, må du angi noen innstillinger for etterfylling i bølgemalen.</span><span class="sxs-lookup"><span data-stu-id="efc6d-124">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="efc6d-125">Bølgemalen må inneholde et bølgetrinn for etterfylling som bare kjøres hvis tildelingen av en vare ikke er vellykket.</span><span class="sxs-lookup"><span data-stu-id="efc6d-125">The wave template should contain a wave step for replenishment that is run only if allocation of an item isn't successful.</span></span> <span data-ttu-id="efc6d-126">Dette bølgetrinnet for etterfylling bruker en bølgetrinnkode for å avgjøre hvilken etterfyllingsmal som skal brukes.</span><span class="sxs-lookup"><span data-stu-id="efc6d-126">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="efc6d-127">I tillegg til å ha et bølgetrinn for etterfylling, må du kontrollere at **etterfylle** er valgt i **Metoder**-delen av bølgemalen.</span><span class="sxs-lookup"><span data-stu-id="efc6d-127">In addition to having a wave step for replenishment, you need to make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span> 

<span data-ttu-id="efc6d-128">Siden **Etterfyllingsmal** inneholder alternativet **Tillat at bølgebehov kan bruke ureserverte antall**.</span><span class="sxs-lookup"><span data-stu-id="efc6d-128">The **Replenishment template** page includes an **Allow wave demand to use unreserved quantities** check box.</span></span> <span data-ttu-id="efc6d-129">Du må merke av for dette alternativet hvis du vil tillate at behovsetterfylling skal trekkes fra ikke-reservert antall fra arbeidet som er generert fra den valgte etterfyllingsmalen.</span><span class="sxs-lookup"><span data-stu-id="efc6d-129">You should select this check box if you want to allow demand replenishment to deduct unreserved quantities from work that is generated from the selected replenishment template.</span></span> <span data-ttu-id="efc6d-130">Hvis du vil aktivere maler for behovsetterfylling til å bruke denne logikken, må du merke av for alternativet for alle eksisterende etterfyllingsmaler.</span><span class="sxs-lookup"><span data-stu-id="efc6d-130">To enable demand replenishment templates to use this logic, select this check box for every existing replenishment template.</span></span> <span data-ttu-id="efc6d-131">Når etterfylling for behov utløses på lageret, vil dette trekke etterspørselen fra eksisterende etterfyllingsarbeid som har ikke-reservert antall, hvis jobben stammer fra etterfyllingsmaler der alternativet **La bølgebehov bruke ureserverte antall** er aktivert.</span><span class="sxs-lookup"><span data-stu-id="efc6d-131">When demand replenishment is triggered in the warehouse, it will deduct the demand from existing replenishment work that has unreserved quantities, if the work originates from replenishment templates where the **Allow wave demand to use unreserved quantities** check box is selected.</span></span>


<span data-ttu-id="efc6d-132">Etterfylling av behov støttes for salgsordrer, overføringsordrer, produksjonsordrer og Kanbaner.</span><span class="sxs-lookup"><span data-stu-id="efc6d-132">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="efc6d-133">Etterfylling basert på minimums- og maksimumsantall</span><span class="sxs-lookup"><span data-stu-id="efc6d-133">Min/Max replenishment</span></span>
<span data-ttu-id="efc6d-134">I etterfylling basert på minimums- og maksimumsantall etterfylles lageret slik at det er mellom minimums- og maksimumsgrensene som er angitt.</span><span class="sxs-lookup"><span data-stu-id="efc6d-134">In Min/Max replenishment, stock is replenished so that it's between the minimum and maximum limits that have been set.</span></span> <span data-ttu-id="efc6d-135">Denne prosessen oppstår vanligvis én gang hver dag for å garantere at alle plukklokasjoner er fylt til det maksimale nivået før plukkingen starter.</span><span class="sxs-lookup"><span data-stu-id="efc6d-135">Typically, this process occurs one time every day to ensure that all picking locations are filled to the maximum level before picking starts.</span></span> 

<span data-ttu-id="efc6d-136">Minimums- og maksimumsantallet angis i en etterfyllingsmal.</span><span class="sxs-lookup"><span data-stu-id="efc6d-136">The minimum and maximum amounts are set in a replenishment template.</span></span> <span data-ttu-id="efc6d-137">Mange av de andre innstillingene i malen ligner på innstillinger i malene som brukes etterfylling basert på bølgebehov.</span><span class="sxs-lookup"><span data-stu-id="efc6d-137">Many of the other settings in the template resemble the settings in templates that are used in Wave demand replenishment.</span></span> <span data-ttu-id="efc6d-138">Malen må inneholde én linje for hver vare og lokasjon.</span><span class="sxs-lookup"><span data-stu-id="efc6d-138">The template should contain one line for each item and location.</span></span> <span data-ttu-id="efc6d-139">Når du kjører etterfylling ved hjelp av den satsvise jobben, evaluerer Finance and Operations om etterfylling kreves i rekkefølgen linjene er organisert i.</span><span class="sxs-lookup"><span data-stu-id="efc6d-139">When you run replenishment by using the batch job, Finance and Operations evaluates whether replenishment is required, in the sequence that the lines are organized in.</span></span> 

<span data-ttu-id="efc6d-140">Vær oppmerksom på at strategien for etterfylling basert på minimums- og maksimumsantall ikke kan etterfylle en tom lokasjon med mindre lokasjonen er definert som fast lokasjon for varen.</span><span class="sxs-lookup"><span data-stu-id="efc6d-140">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="efc6d-141">Hvis lokasjonen som skal etterfylles ikke er en fast lokasjon, er det ikke mulig å fastslå hvilken vare som skal etterfylles.</span><span class="sxs-lookup"><span data-stu-id="efc6d-141">If the location that must be replenished isn't a fixed location, it's not possible to determine which item to replenish.</span></span> <span data-ttu-id="efc6d-142">Derfor kreves det minst noe beholdning før etterfylling oppstår.</span><span class="sxs-lookup"><span data-stu-id="efc6d-142">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
